# 宋言的工房 (SUNGYAN-workshop)

穩定中的劇震——言織 v0.1 開發記錄。

這是一個使用 Next.js 14 (App Router) 建立的純靜態個人網站，專為 Cloudflare Pages 設計。所有內容均為 Markdown 檔案，無資料庫，無動態伺服器。

## 技術棧

- 框架：Next.js 14 (App Router, 靜態匯出 `output: 'export'`)
- 樣式：Tailwind CSS + 純 CSS 特效
- 內容：Markdown + `gray-matter` 解析
- 部署：Cloudflare Pages (透過 Git 整合)
- API：Cloudflare Pages Functions (用於管理後台發布)

## Cloudflare Pages 部署指南

**絕對不要使用「直接上傳」功能。** 必須透過 Git 推送觸發自動構建。

1. 登入 [Cloudflare 儀表板](https://dash.cloudflare.com/)，進入 **Workers & Pages**。
2. 點擊 **Create application** -> **Pages** -> **Connect to Git**。
3. 授權 GitHub，並選擇此倉庫 `yanzhi-protocol/SUNGYAN-workshop`。
4. 進行以下構建設定：
   - **Framework preset**: `Next.js`
   - **Build command**: `npm run build`
   - **Build output directory**: `out`
5. **環境變數設定 (Environment variables)**：
   展開環境變數區塊，新增以下三個變數：
   - `ADMIN_SECRET`: (自訂一個後台登入密碼)
   - `GITHUB_TOKEN`: `(填入你的 GitHub Personal Access Token)`
   - `GITHUB_OWNER`: `yanzhi-protocol`
   - `GITHUB_REPO`: `SUNGYAN-workshop`
6. 點擊 **Save and Deploy**。
7. 部署完成後，你會獲得一個 `*.pages.dev` 連結。

## 管理後台使用方式

1. 進入 `你的網域/admin`。
2. 輸入你在 Cloudflare Pages 設定的 `ADMIN_SECRET` 解鎖。
3. 填寫標題、分類、標籤與 Markdown 內容。
4. 點擊「發布文章」，Cloudflare Pages Function 會透過 GitHub API 將 Markdown 檔案直接推送到倉庫的 `content/logs/` 目錄。
5. GitHub 倉庫更新後，Cloudflare Pages 會自動觸發重新構建，約 1~2 分鐘後網站內容即會更新。

## 本地開發

如果你需要在本地預覽：

```bash
npm install
npm run dev
```

> 注意：本地環境無法測試 `/api/commit` 發布功能，因為它依賴 Cloudflare Pages Functions 環境。
